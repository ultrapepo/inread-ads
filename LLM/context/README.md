# Context Files

This directory holds feature design documents created by the Orchestrator.

Each file documents the requirements, data schemas, command flows, and edge cases for a feature BEFORE the Coding LLM implements it.

## Naming Convention
`{feature-name}.md` — e.g., `user-authentication.md`, `payment-processing.md`

## Purpose
- Serves as the "truth document" for what a feature should do
- Referenced by the handoff prompt
- Referenced during audits to verify correctness
